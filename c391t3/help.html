Installation Guide:
  1. Move untarred project folder to your catalina webapps folder
  2. Run "make" on the Makefile in the main project folder
  3. Run "@dbFiles/setup.sql" in sqlplus
  4. (Optional) run your own insert file or use "@dbFiles/insert.sql" to populate database
  5. (Re)start your tomcat server
  6. Login at (server location):(Port)/c391t3/login.html
You can also teardown our custom database additions by running "@dbFiles/drop.sql" in sqlplus.

Logging In:

The login page allows a user to log in with a given userName and password.
Also, if the "Use a different Database" checkbox is checked, the user can
specify a specific database userName and password to log into a different database.

Main Page:

The main page of our website contains a header that shows if a user is logged in and
a sidebar that contains links to every module that the user has permission to access.
You can also click the logout link in the header to log out of the website securely.
In the next section we will talk about the areas of the website that everyone can access.

Edit Profile:

A user can choose to edit their profile by selecting the "Edit Profile" link in the sidebar
The edit profile page display the users current information for editing, as well as the
option to change their password. If the "Change Password" checkbox is checked, then the
user needs to enter their current password and a new password. If the old password is
incorrect, the password will not be changed, and the system will warn the user.

Searching:

The search module allows the user to search through the records available to them using a
number to parameters:

  • Keywords: Display only the records that contain any element of the comma-separated keyword list
  • Start Date: Only view records after this date
  • End Date: Only view records before this date
  • Order By: Order the results by rank, most-recent-first or most-recent-last
To order by rank, some keywords must be given. After the parameters are given, a table will be
displayed containing the results of your search. The table consists of all relevant record infomation
inlcuding any images that are associated with the record. The associated images are located in the
row after the record. The thumbnails of the images can be selected and the full image will display in
another window.

Administration Modules

These modules are only accessible by an administrator.

User Management:

Upon entering the User Management module, you can either create a new user, or modify an existing user.
If you choose to create a new user, you can enter their personal information, along with a username and
password. After creating a new user, the user is added to the list of existing users. Choosing to edit
an existing user will allow you to update their personal information, such as name, address, and password.
Also, if the user is a patient, you will get a list of family doctors for that patient, and be able to add
and remove doctors. If the user is a doctor, you will get a list of patients that the doctor oversees, and
be able to add and remove patients. This will affect which records the doctor can view.

Report Generator:

The report generator allows an admin to view a list of patients that have been diagnosed with a certain
disease over a period of time. For this module to work, you must enter a diagnosis, start and end date, and
you will be warned if you do not. The report generator will then return a list of patients that fit the
criteria, including their name, address, phone number, and test date they were first diagnosed.

Back To Top

Data Analysis (OLAP):

The Data Analysis Module allows an admin to generate OLAP reports for data analysis. The admin has the
choice of grouping the data by patient, test_type, and/or time. The time can be drilled down from years to
months and weeks. After the admin has chosen at least one parameter, we display the OLAP data table. The table
contains the criteria you requested along with the number of records that fit the criteria.

Radiologist Modules

These modules are only accessible by a radiologist.

Record Uploading:

From the Record Uploading module, you can add a new record by filling in the form. If the information is
valid, then the page will display the record id and allow you to add images to the newly created record.

Back To Top

Image Uploading:

Either from the main record uploading page or after successfully adding an image or record,
you can upload an image associated with a record you created. If you are uploading from the base
Record Uploading module, you need to specify the record id and the image you want to add. You can
only add images to a record that you entered into the database. If you are uploading after a successful
upload, you do not need to reenter the record id, only the image to upload. Every image uploaded will
show up in the search results.
